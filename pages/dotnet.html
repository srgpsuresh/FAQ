1. What is Dependency Injection?

Category: .NET Core

Difficulty: Beginner

Definition

Dependency Injection (DI) is a design pattern used to achieve loose coupling by injecting dependencies instead of creating them inside the class.

Why do we use it?

  • Loose Coupling
  • Easy Unit Testing
  • Maintainability
  • Scalability

Real-Time Example

Employee Management System EmployeeController ↓ EmployeeService ↓ EmployeeRepository ↓ SQL Server

Architecture


                    Controller
                          │
                          ▼
                    Service
                          │
                          ▼
                    Repository
                          │
                          ▼
                    Database

Execution Flow

  1. Client sends HTTP Request
  2. Controller receives request
  3. DI injects Service
  4. Service executes Business Logic
  5. Repository accesses Database
  6. Response returned to Client

C# Example



public class EmployeeController
{
    private readonly IEmployeeService service;

    public EmployeeController(
        IEmployeeService service)
    {
        this.service = service;
    }
}

Advantages

  • Reusable
  • Testable
  • Easy Maintenance
  • Follows SOLID Principles

Disadvantages

  • Learning Curve
  • Extra Configuration

Performance

Built-in DI container is optimized and suitable for enterprise applications.

Interview Answer (30 Seconds)

Expand Answer

Dependency Injection is an IoC pattern used to inject dependencies through constructors, making applications loosely coupled, testable, and maintainable.
