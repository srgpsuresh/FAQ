API Design - Top 10 Interview FAQs

1. What is Resource-Oriented API Design?
Definition
Design APIs around business resources (nouns) instead of actions (verbs).
Real-Time
Use /api/orders/101 instead of /GetOrderById.
2. What are REST API Naming Conventions?
Definition
Use plural nouns, lowercase URLs, and meaningful resource names.
Real-Time
/api/customers, /api/products
3. Which HTTP Method should be used?
Definition
GET for Read, POST for Create, PUT for Update, PATCH for Partial Update, DELETE for Remove.
Real-Time
Create Employee using POST /api/employees.
4. Why are HTTP Status Codes important?
Definition
Status codes indicate the result of an API request.
Real-Time
Return 201 after creating an order, 404 when employee not found.
5. Why use JSON?
Definition
JSON is lightweight, readable and platform independent.
Real-Time
Angular exchanges JSON with ASP.NET Core APIs.
6. What is API Versioning?
Definition
Allows multiple API versions without breaking existing clients.
Real-Time
/api/v1/products and /api/v2/products
7. Why use DTO?
Definition
Transfer only required data between client and server.
Real-Time
Hide Password and Salary fields.
8. Why validate requests?
Definition
Prevent invalid data from entering the application.
Real-Time
Reject registration if Email is invalid.
9. Why use Pagination?
Definition
Improve performance by returning smaller result sets.
Real-Time
/api/products?page=1&pageSize=20
10. What is Idempotency?
Definition
Executing the same request multiple times produces the same result.
Real-Time
Multiple PUT requests update the same employee without creating duplicates.
